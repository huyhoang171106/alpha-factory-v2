# Team Sharing Playbook (Private Repo)

## Access Model
- Keep repository private.
- Grant least privilege:
  - `Admin/Maintain`: core owners only
  - `Write`: trusted contributors
  - `Triage/Read`: reviewers and observers

## Branch and Review Policy
- Protect `main`:
  - Require PR before merge
  - Require checks: `CI`, `Security Audit`
  - Require CODEOWNER review
  - Disable force push and branch deletion
- Note: branch protection on private repos requires Pro/Student (or public repo on Free).

## Secrets and Runtime Isolation
- Store credentials in GitHub Secrets only:
  - `WQ_EMAIL`, `WQ_PASSWORD`, optional `OPENROUTER_API_KEY`
- Keep local `.env` personal; do not sync `.env` across teammates.
- For production automation, create GitHub Environment (e.g., `prod`) and scope secrets there.

## Shared Operations
- Preferred entrypoint for automation:
  - `python alpha_factory_cli.py auto --profile gha --skip-install`
- Recovery commands:
  - `python alpha_factory_cli.py replay-dlq --limit 50`
  - `python alpha_factory_cli.py sync-submit --limit 40`
  - `python alpha_factory_cli.py kpi --minutes 60`

## Public Profile Strategy
- Publish only sanitized report generated by:
  - `python alpha_factory_cli.py public-report --minutes 60 --out results/public_report.json`
- Never publish internal expressions, alpha IDs, or credentials.
